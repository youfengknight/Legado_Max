# Skill 1 标准版（通用万能，直接复制）
## 技能名称
代码修改全流程合规校验
## 技能描述
每次根据用户需求完成代码编写、修改、迭代后，**自动执行全维度自检**，无需用户额外指令，完成全部校验并输出完整检测报告，严格约束代码修改范围、设计规范、日志完整性、Bug隐患与项目运行可行性。
## 触发条件
每次完成用户提出的代码需求、代码修改、功能增补、代码优化任务后，自动触发本技能全部检查流程。
## 执行规则
1. **修改范围校验**
    严格核对本次修改内容，仅改动需求指定的代码、函数、文件；禁止修改无关业务代码、配置、常量、其他无关函数、第三方调用、工具类通用逻辑；核对代码差异，确认无额外冗余修改、无无关代码改动、无隐性修改。
2. **单一职责原则校验**
    检查所有修改后的类、函数、方法：一个函数仅实现单一功能，不混合多业务逻辑、不耦合多余职责、不臃肿过长；无多功能堆砌、无跨职责逻辑、无冗余嵌套；职责边界清晰，不符合则自动重构优化。
3. **日志完整性校验**
    所有新增、修改的业务逻辑、分支流程、异常捕获、入参出参、关键节点必须添加规范日志；包含正常流程日志、入参出参日志、异常捕获日志、错误堆栈日志；日志级别合理、信息完整可读，禁止缺失日志、裸异常、无日志捕获。
4. **Bug与运行可行性校验**
    自动检测修改代码的语法错误、空指针、边界异常、参数异常、逻辑漏洞、潜在Bug；校验代码语法正确性、依赖调用完整性；验证修改代码可在原项目环境正常运行，不影响原有功能、不引入兼容性问题，给出运行可行性结论。
5. **输出规范**
    完成修改后统一输出：修改内容说明、四项自检完整报告、问题修复记录、最终合规可用代码。

# Skill 2 严格高级版（最强约束，企业开发专用，最推荐）
## 技能名称
AI代码修改全自动合规审核技能
## 技能描述
专属自动化代码审核技能，在每一次完成用户代码需求、代码修改、功能开发完毕后，**无条件自动执行全流程深度检测**，全程自主校验、自主整改问题，最终输出完全合规、可直接合并提交的代码，并附带完整审核报告。
## 触发条件
每当完成任意代码编写、修改、迭代、优化任务结束后，自动启动全部校验流程。
## 详细执行指令
### 1. 修改范围锁定检测
1. 仅针对用户明确提出的需求代码进行修改，严格限定修改文件、修改代码块范围
2. 禁止改动任何非需求相关代码、全局配置、公共工具方法、其他业务模块、原有稳定逻辑
3. 核对代码差异增量，确认无擅自修改、无额外改动、无隐性代码变更、无多余删减代码
### 2. 单一职责原则深度校验
1. 所有修改/新增的函数、类严格遵循单一职责：**一个方法仅负责一件独立业务功能**
2. 禁止函数功能臃肿、多逻辑耦合、跨业务职责混合、无关逻辑内嵌
3. 若职责不清晰、逻辑混杂，自动拆分重构代码，保证结构简洁、职责单一
### 3. 标准化日志强制校验
1. 所有关键执行节点、流程分支、数据处理、接口调用必须添加规范业务日志
2. 异常捕获块必须完整记录异常信息、堆栈、上下文参数日志
3. 日志区分正常信息、警告、错误级别，信息完整可追溯，不允许无日志、裸抛异常、无异常记录
### 4. 代码Bug与项目运行检测
1. 静态检测全部潜在Bug：语法错误、参数错误、空值风险、边界问题、逻辑漏洞、资源泄漏
2. 校验代码依赖引用完整，无缺失调用、无废弃API误用
3. 验证修改代码兼容原有项目整体架构，接入项目可正常运行，不破坏原有功能，无回归问题
### 5. 最终输出要求
先输出本次**完整自检审核报告**（含4项检测结果、问题、修复点），再输出最终定稿代码，所有不合格项必须全部自主修复完毕再交付。
