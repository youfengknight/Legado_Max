### 五、架构模式分析

项目采用的是 **MVVM (Model-View-ViewModel)** 架构模式，具体体现：

1. **View层** (`ui/` 包): Activity + Fragment + ViewBinding
   - 所有 Activity 继承自 `BaseActivity<VB>` 或 `VMBaseActivity<VB, VM>`
   - 使用 ViewBinding 进行视图绑定（在 build.gradle 中启用了 `viewBinding = true`）
2. **ViewModel层** (`ui/` 各子包中的 ViewModel):
   - 所有 ViewModel 继承自 `BaseViewModel`（继承 `AndroidViewModel`）
   - `BaseViewModel` 封装了协程执行方法 (`execute`, `executeLazy`, `submit`)
   - 例如: `ReadBookViewModel`, `SearchViewModel`, `BookSourceViewModel` 等
3. **Model层** (`model/` + `data/` + `help/` 包):
   - `data/` 包含 Room 数据库、DAO、实体类
   - `model/` 包含业务逻辑（规则解析、书籍获取、阅读控制等）
   - `help/` 包含配置管理、网络帮助等
4. **事件总线**: 使用 `LiveEventBus` 进行组件间通信
5. **数据存储**: Room 数据库 (版本15，含21个DAO)，SharedPreferences

## 五、学习建议

1. **先跑起来**：先把项目编译运行，实际操作一遍所有功能，建立感性认识
2. **从入口追踪**：从 `App.kt` → `WelcomeActivity` → `MainActivity` 跟一遍启动流程
3. **挑一个功能走通**：比如"搜索一本书"的完整流程：`SearchActivity` → `SearchViewModel` → `WebBook` → `AnalyzeRule` → 数据库存储
4. **重点看规则引擎**：`model/analyzeRule/` 是整个项目最核心、最独特的部分，理解了它就理解了书源系统
5. **善用包内 README**：项目每个包下都有 README.md，可以帮助你快速了解该包的职责
6. **看规则帮助文档**：`assets/web/help/md/` 下有书源规则、JS帮助等文档，是理解规则系统的第一手资料

***

**总结**：这个项目最值得深入学习的是 **MVVM架构实践**、**规则解析引擎（analyzeRule）** 和 **Kotlin协程的运用**。建议从启动流程入手，再选一个你感兴趣的功能模块深入追踪，最后攻克规则引擎。
